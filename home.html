Part 1: Identifying Usability Problems

Picking a Website

For this project, I decided to redesign the "Featured Books" webpage for a local Providence book store called Books on the Square. Here is a screenshot of this specific page of the company's website:



Finding Problems

The first problem that I found when looking at this interface was the confusing layout. The header at the top of the page is large but not used for navigation, and the navigation bar below extends down for almost half of page. There are too many navigation options to choose from, and the actual content of the page, "Featured Books," can barely be seen when the page loads. Scrolling down, we can see that the input area for searching for books is hidden on the right, amongst other information about logging in and contacting the store. On the left are the featured books, with cover photos listed under headings of specific categories of featured books. These issues can be summarized in the following list:

Usability:

  • It is difficult to know where to look first on the page - your attention is draw to many different places at once.
  • It is not immediately clear that this is indeed the featured books page, since the content is not clearly displayed.
  • Due to the confusing layout hierarchy there is no clear visual flow, which makes the page hard to read and follow.
  • Simplicity is important in usability; unfortunately, this web page is quite complex and convoluted.

Learnability:
  • It is hard to learn what interactions are expected on this page: is the main focus to login? To shop? To view genreal and specific information about the store and how it operates?
  • The page is split into two columns the whole way down, and it takes time to realize what the dedicated functions of these two columns are (the left column contains the featured book information; the right column is a random assortment of navigation, subscription, purchase, and search information).
  • This is a difficult web page to skim, and the user must put time into learning how to operate it.
  • Grouping helps first-time users determine how to navigate a web page; however, this page only groups together unrelated functionalities and thus confuses the user.

Memorability:
  • It is difficult to remember how to navigate to a certain part of the website because there are so many options.
  • After being overwhelmed by the sheer number of elements on the screen, it is difficult for the user to remember why they navigated to this page in the first place.
  • It is hard to find and remeber patterns of use, since things that should be normalized - such as alignment, color, and typography - are not.

Accessibility

Next, I used the WebAIM WAVE tool to detect any possible accessibility concerns. This analysis showed that there are 24 contrast errors on this web page, 3 empty buttons, 3 images without alternative text, and 1 missing form label. It also noted that there are 79 alerts, many of which refer to suspicous alternative text. Further analysis using the screen reader showed that a lot of important identifying information is not captured with assisted viewing: the entire top panel, which includes the name of the store and its motto, location, and contact information, is a single image. This information is therefore not picked up by the screen reader. Additionally, it should be noted that after looking at this image, the screen reader looks through all of the navigation bar items and all of the listed featured books - the user is not shown the right column of information (including login options, search capabilities, and purchase information) until all of the books have been looked through.

Part 2: Visual Redesign

Low-fidelity Wireframing: Balsamiq

The annotations on the wireframes below focus on several main changes:

  • Making the navigation bar more clear and concise.
  • Ensuring that the store's logo and motto are eye-catching and strategically placed.
  • Having the store's contact information be very visible the entire time the user is on the web page.
  • Displaying featured books in a clear way that is effective and yet not overwhelming.
  • Separating different featured book categories into distinct panels.
  • Ensuring that the only information present is what was the user expected to see when clicking on the "books" item n the navigation menu.





High-fidelity Prototyping: Figma

The annotations on the prototypes below focus on several main changes:

  • Making the navigation bar more clear and concise, and hiding extra items when the sceen size is small.
  • Ensuring that the store's logo and motto are eye-catching and strategically placed no matter what device the user is on. This is achieved using flexbox.
  • Having the store's contact information be very visible the entire time the user is on the web page, using a sticky footer.
  • Displaying featured books in a clear way using flexbox to ensure that content is shifted according to the user's device.
